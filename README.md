## Setup

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
```

## Run Prediction (claims.csv → output.csv)

```bash
python code/main.py
```

- Reads `dataset/claims.csv` (44 rows)
- Loads `dataset/user_history.csv` and `dataset/evidence_requirements.csv`
- Sends 1 Claude API call per claim (with all images attached)
- Writes `output.csv` incrementally (safe to resume)
- Uses 2s inter-call delay to respect rate limits

## Run Evaluation (sample_claims.csv → evaluation_report.md)

```bash
python code/evaluation/main.py
```

- Runs the full pipeline on `dataset/sample_claims.csv` (20 labeled rows)
- Computes field-level accuracy vs ground truth
- Writes `code/evaluation/evaluation_report.md`

## Architecture

```
claims.csv
    ↓
[Preprocessor]          No API calls
  - Adversarial detection
  - User risk scoring (user_history.csv)
  - Evidence requirement lookup (evidence_requirements.csv)
  - Image loading + base64 encoding
    ↓
[LLM Agent]             1 API call per claim
  - System prompt with decision rules
  - Claim context + user history text
  - Evidence requirement text
  - All images attached as vision blocks
  - Returns structured JSON
    ↓
[Postprocessor]         No API calls
  - Validate against allowed value lists
  - Enforce business rules
  - Merge pre-computed risk flags
    ↓
output.csv
```

## Key Design Decisions

### One API Call Per Claim
All images for a claim are sent in a single multi-modal message. This minimises cost and latency while giving the model full context across all submitted photos.

### Adversarial Detection Pre-Filter
Prompt-injection attempts in the claim text are detected heuristically before the API call. The LLM prompt is also instructed to flag and ignore such content. The `text_instruction_present` risk flag is added deterministically.

### User History as Context Only
User risk scores are computed before the API call and injected as context text. They cannot override a clear visual verdict — they only add risk flags and enrich the justification.

### Streaming Writes
`output.csv` is written one row at a time (append mode) so that a partially-completed run is not lost if the process is interrupted.

### Resume Support
Pass `start_from=N` to `run_pipeline()` to resume from row N after an interruption.
