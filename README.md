## Setup

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
```

## Run Prediction — `claims.csv` → `output.csv`

```bash
# Default: Strategy B (Chain-of-Thought, best performer from evaluation)
python code/main.py

# Explicit strategy selection
python code/main.py --strategy A     # Direct Classification
python code/main.py --strategy B     # Chain-of-Thought Reasoning

# Resume an interrupted run from row N
python code/main.py --resume 10
```

Output is written **incrementally** (one row at a time in append mode) so the
file is never fully lost if the process is interrupted.

## Run Evaluation — `sample_claims.csv` → `evaluation_report.md`

```bash
python code/evaluation/main.py
```

This runs **both Strategy A and Strategy B** on all 20 labeled sample rows,
computes field-level accuracy for each, compares them head-to-head, and
writes `code/evaluation/evaluation_report.md`.

## Architecture

```
claims.csv
    │
    ├─► [Preprocessor]          ← No API calls
    │     Adversarial text detection
    │     User risk scoring (user_history.csv lookup)
    │     Evidence requirement lookup (evidence_requirements.csv)
    │     Image loading + base64 encoding
    │
    ├─► [LLM Agent]             ← 1 API call per claim
    │     Strategy A: Direct Classification (concise prompt)
    │     Strategy B: Chain-of-Thought (6-step reasoning)
    │     All images sent in one multi-modal message
    │     3 retries with backoff on failure
    │
    ├─► [Postprocessor]         ← No API calls
    │     Validate against allowed value lists
    │     Enforce business rules (evidence=false → not_enough_info)
    │     Merge pre-computed risk flags
    │
    └─► output.csv              ← Streamed row by row
```

## Two Prompting Strategies

### Strategy A — Direct Classification
A concise single-pass prompt (~400 token system prompt). Provides the model
with the complete allowed-value lists and hard decision rules, then asks it to
inspect the images and immediately output a structured JSON verdict.

**Best for**: Clear-cut claims with good image quality.
**Trade-off**: Less explicit reasoning on ambiguous or adversarial cases.

### Strategy B — Chain-of-Thought Reasoning *(selected for output.csv)*
A six-step guided reasoning prompt (~600 token system prompt). The model is
walked through: visual inventory → claim extraction → evidence sufficiency
check → verdict → risk flags → severity, before emitting JSON.

**Best for**: Adversarial claims, multi-language transcripts, ambiguous damage.
**Trade-off**: Slightly higher token usage (~200 tokens/call) and latency.

Strategy B was selected as the final strategy based on evaluation results
(see `code/evaluation/evaluation_report.md`).

## Key Design Decisions

| Decision | Rationale |
|---|---|
| One API call per claim | All images sent in a single multi-modal message; minimises cost and latency |
| Adversarial pre-filter | Heuristic catches injection before the API call; model prompt also independently instructs ignoring directives |
| User history as context only | Computed pre-call and injected as text; cannot override a clear visual verdict |
| Streaming output writes | Append mode means partial runs are never fully lost |
| Business rules post-LLM | `evidence_standard_met=false` → `not_enough_information` enforced in code, not left to the model |
| Resume support | `--resume N` skips already-processed rows on restart |
